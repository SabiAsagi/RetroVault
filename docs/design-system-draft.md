# RetroVault 디자인 시스템 초안

이 문서는 Figma 리디자인과 웹/모바일 UI 개선을 위한 디자인 시스템 초안이다. 목표는 레트로 게임 아카이브의 개성을 유지하면서도, 검색과 비교, 컬렉션 관리가 편한 실사용 UI를 만드는 것이다.

## 디자인 방향

- 레트로 무드는 색상, 작은 픽셀 포인트, 카드 질감으로 표현한다.
- 본문과 데이터 영역은 가독성을 우선한다.
- 모바일에서는 탐색 속도와 터치 편의성을 가장 중요하게 본다.
- 관리/요청/검색 화면은 장식보다 정보 밀도와 상태 구분을 우선한다.

## 색상 토큰

### 기본 배경

- `vault-bg`: 전체 배경
- `vault-surface`: 카드, 패널, 목록 영역
- `vault-surface-light`: hover, 선택, skeleton 배경
- `vault-border`: 기본 경계선
- `vault-border-light`: 강조 경계선

### 포인트 색상

- `mint`: 주요 액션, 성공, 활성 상태
- `neon-blue`: 게임/정보 링크, 탐색 강조
- `neon-purple`: 플랫폼/세대 강조
- `amber`: 인기, 평점, 제작사 강조
- `coral`: 경고, 반려, 신고, 삭제

### 텍스트

- `text-primary`: 제목과 핵심 정보
- `text-secondary`: 설명과 보조 정보
- `text-muted`: 메타 정보, 빈 상태 안내

## 타이포그래피

- 기본 폰트: Pretendard
- 브랜드 포인트: Press Start 2P
- 픽셀 폰트는 로고, 짧은 라벨, 모드 이름처럼 짧은 텍스트에만 사용한다.
- 목록/상세/관리 화면에서는 본문 가독성을 위해 일반 산세리프를 사용한다.

## 공통 컴포넌트

### Navigation

- 데스크톱: 상단 글로벌 검색 + 2단 메뉴
- 모바일: 상단 검색 + 하단 탭 5개
- 하단 탭: 홈, 게임, 플랫폼, 커뮤니티, 타임라인

### Search Dropdown

- 결과 그룹: 게임, 플랫폼, 회사, 유저, 공개 컬렉션
- 각 항목은 아이콘/이미지, 제목, 보조 텍스트를 가진다.
- 결과 없음 상태는 짧은 문장으로만 안내한다.

### Game Card

- 커버 이미지
- 제목
- 플랫폼
- 출시연도
- 장르/인기도/조회수 중 화면 맥락에 맞는 보조 정보
- 컬렉션 추가 상태 표시

### Platform Card

- 기기 이미지 또는 아이콘 placeholder
- 플랫폼명
- 제조사
- 출시연도
- 세대/타입
- 연결 게임 수

### Company Card

- 로고 또는 placeholder
- 회사명
- 개발/유통 구분
- 국가
- 대표작 또는 연결 게임 수

### Collection Card

- 컬렉션 제목
- 소유자
- 대표 게임 커버 3개
- 좋아요/조회수
- 공개/비공개 상태

### Admin Status Badge

- `PENDING`: amber
- `APPROVED`: mint
- `REJECTED`: coral
- `BLOCKED` 또는 `BANNED`: coral 강조

## 모바일 앱 프로토타입 화면 범위

1. 홈
2. 게임 검색/목록
3. 게임 상세
4. 플랫폼 탐색
5. 내 컬렉션
6. 컬렉션 추가/수정 모달
7. 유저 컬렉션 탐색
8. 프로필
9. 알림/요청 상태
10. 설정

## Figma 작업 순서

1. 색상/텍스트 스타일 변수 생성
2. 버튼, 입력창, 배지, 카드 기본 컴포넌트 제작
3. 게임 카드, 플랫폼 카드, 회사 카드 제작
4. 데스크톱 홈/게임 목록 리디자인
5. 모바일 하단 탭 기준 주요 화면 제작
6. 프로토타입 연결
7. 실제 웹 화면과 비교해 간격과 상태를 보정

## 점검 기준

- 모바일 360px 폭에서 탭과 버튼 텍스트가 잘리지 않는가?
- 검색 결과가 5개 그룹으로 나뉘어도 읽기 쉬운가?
- 카드 이미지가 없을 때 placeholder가 자연스러운가?
- 관리자 상태와 사용자 상태가 색상만이 아니라 텍스트로도 구분되는가?
- 레트로 효과가 정보 탐색을 방해하지 않는가?
